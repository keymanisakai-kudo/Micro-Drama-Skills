# build-story-bible

## Purpose

基于已初始化的项目元数据，生成 `script/full_script.md`，建立整部作品的叙事基础：
- 世界观设定（200-300字）
- 总体故事大纲（500字）
- 25集各集概要（每集含剧情摘要、关键事件、情感基调）

本技能产出是后续角色/场景/道具设计（`build-production-bibles`）和逐集内容生成（`build-episode-pack`）的唯一叙事来源。

---

## When to Use

- 项目已初始化（`metadata.json` 存在），剧本尚未编写（`script/full_script.md` 不存在）
- 用户要求重新编写剧本（覆盖现有 `full_script.md`）

---

## When Not to Use

- `metadata.json` 尚不存在，请先执行 `init-drama-project`
- 只需补充某集剧情摘要，而非重写整部剧本

---

## Inputs

| 输入 | 来源 | 必填 |
|------|------|------|
| `metadata.json` | `projects/{project_id}_{slug}/metadata.json` | 是 |
| 作品类型、风格（来自 metadata） | 文件读取 | 是 |
| 用户附加要求（如"主角3人"、"科幻背景"） | 用户请求 | 否 |

---

## Outputs

| 产物 | 路径 | 说明 |
|------|------|------|
| `full_script.md` | `projects/{project_id}_{slug}/script/full_script.md` | 含世界观、大纲、25集概要 |

### full_script.md 结构

```markdown
# 《作品名称》完整剧本

## 作品信息
- **类型**：[冒险/奇幻/科幻/日常/恋爱 等]
- **风格**：[热血/治愈/悬疑/搞笑 等]
- **视觉风格**：[风格预设名称，如 Cinematic Film]
- **目标受众**：[少年/少女/青年/全年龄]
- **总时长**：25集 × 30秒 = 12分30秒
- **核心主题**：一句话概括

## 世界观设定
[200-300字描述]

## 故事大纲
[500字总体故事线]

## 各集概要
### 第1集：[标题]
- **剧情概要**：[50字]
- **关键事件**：[列表]
- **情感基调**：[喜/怒/哀/乐/紧张/温馨]

...（共25集）
```

---

## Constraints

### 叙事结构约束

| 集数段 | 叙事功能 |
|-------|---------|
| EP01–EP03 | 世界观介绍、角色登场、引入核心冲突 |
| EP04–EP08 | 冲突升级、角色关系建立 |
| EP09–EP15 | 高潮前奏、多线叙事、伏笔布局 |
| EP16–EP20 | 高潮阶段、转折、揭示 |
| EP21–EP24 | 最终决战或情感爆发 |
| EP25 | 结局、余韵 |

### 剧本质量约束

1. **故事完整性**：25集须有完整的起承转合，不得在中途戛然而止
2. **每集30秒约束**：每集聚焦一个核心场景/事件，信息密度高，不得铺陈过多
3. **上下结构预设**：每集概要应隐含「上半部分铺垫/展开、下半部分高潮/转折」的双段结构，便于后续分镜创作
4. **结尾钩子**：每集结尾须留悬念或情感钩子，带动连续观看
5. **集数概要长度**：每集概要约50字，不超过100字

## Workflow

### Step 1：读取元数据

读取 `metadata.json`，确认 `project_id`、`project_name`、`genre`、`style`、`visual_style`、`core_theme`。

### Step 2：构建世界观和大纲

- 撰写世界观设定（200-300字）
- 撰写故事大纲（500字），明确主线冲突和最终走向
- 确保故事弧度符合叙事结构约束中的集数分段规划

### Step 3：逐集生成概要

- 按 EP01–EP25 顺序，为每集写出：
  - 集标题
  - 剧情概要（50字）
  - 关键事件列表（2-3项）
  - 情感基调

### Step 4：写入 full_script.md

按照 outputs 中定义的结构写入文件。

- [ ] `metadata.json` 已读取，作品基本信息已确认
- [ ] 世界观设定已写入（200-300字）
- [ ] 故事大纲已写入（500字）
- [ ] 所有 25 集概要已写入
- [ ] 叙事弧度符合 EP01-03/04-08/09-15/16-20/21-24/25 分段规划
- [ ] 每集结尾含悬念或情感钩子
- [ ] `full_script.md` 文件已写入正确路径

---

## Failure Modes

| 错误场景 | 处理方式 |
|---------|---------|
| `metadata.json` 不存在 | **阻断执行**，提示先运行 `init-drama-project` |
| `full_script.md` 已存在 | 警告用户将覆盖现有剧本，等待确认后继续 |
| 25集概要数量不足 | 自动补全至25集，不得以少于25集的剧本作为最终输出 |
| 某集缺少情感基调 | 根据剧情上下文自动推断，不得留空 |
